# IBM-HR-Analytics-Predict
📌 1. 项目背景 (Project Background)
在现代企业管理中，核心人才的流失（Attrition）会给公司带来巨大的招聘成本和知识损失。本项目基于 IBM HR 真实数据集，利用 Python 进行探索性数据分析（EDA）并构建机器学习模型，旨在：

量化影响员工留失的关键指标。

预测具有高流失风险的员工群体。

提供基于数据驱动的留才建议，将 HR 工作从“经验驱动”转向“数据驱动”。

🛠️ 2. 技术栈 (Technical Stack)
数据处理: Pandas, NumPy

可视化: Matplotlib, Seaborn (用于热力图、分布图分析)

机器学习: Scikit-learn (包含 RandomForestClassifier 随机森林模型)

环境工具: VS Code, Jupyter Notebook

📊 3. 核心分析流程 (Project Pipeline)
数据清洗与预处理:

处理非数值型特征（如：出差频率、部门、加班状态）。

汉化关键特征标签，提升分析报告的可读性。

探索性数据分析 (EDA):

绘制相关性热力图，识别特征间的共线性。

分析“月收入”、“加班”、“年龄”与离职率的直接关系。

模型构建与优化:

使用 随机森林 算法进行分类预测。

针对数据不平衡（离职人数远少于在职人数）进行了逻辑优化，确保模型对离职信号更敏锐。

📈 4. 关键洞察与商业价值 (Insights & Business Value)
模型表现: 最终预测准确率达到 86.39%。

核心驱动因素排名:

月收入 (30.2%): 低薪区间员工流失风险极高。

年龄 (19.0%): 年轻员工在职业初期更易波动。

是否加班 (17.7%): 经常加班的员工流失概率是非加班者的 3 倍。

商业价值测算: 在测试样本中，模型准确识别出潜在离职员工。根据行业平均招聘成本估算，若通过精准干预留住 30% 的高风险人才，预估可为公司节省约 $64,379 的人才置换成本。

⚠️ 5. 技术攻关与故障排查 (Engineering Challenges)
在项目开发阶段，遭遇了大规模的 NumPy 2.x 与 Scikit-learn/Pandas 环境兼容性冲突（导致 ImportError: numpy.core.multiarray failed to import）。

应对方案: 我独立进行了环境诊断，放弃了不稳定的第三方解释库（如 SHAP），通过强制降级 NumPy 版本并采用 原生 Python 列表处理技术 重构了特征重要性分析逻辑。

收获: 这次经历显著提升了我对 Python 依赖管理（Dependency Management）的理解，证明了在生产环境崩溃时快速寻找“替代路径”的工程应变能力。
